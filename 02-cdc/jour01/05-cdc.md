# cahier des charges

- pour tout projet informatique, il y a quand même un minimum de documentation à donner à l'équipe développeur pour pouvoir réaliser le projet 

- le cahier des charges 
- document word entre 10 et 40 pages qui décrit le besoin du client 

-  3 parties :
    - présenter les intervenants client / prestataire
        - client : présenter / siège social / secteur d'activité
        - le prestataire : raison social / coordonnées / 
        - les objectifs du clients et du projet
    - analyse marketing / etude de marché 
        - Facultative lorsque l'on lance un nouveau site internet 
        - Etude de marché des principaux concurrents du site que l'on va créer 
        - Décrire le profil des utilisateurs types de notre site "Persona"
    - spécifications techniques
        - la partie la plus longue du document
        - décrire techniquement le besoin du client 
        - Arborescence = liste de toutes les pages du site internet sous forme de graph 
        - Charte graphique = couleurs principales du site + le logo 
        - La liste des technologies utilisées : 
            - Wordpress / Prestashop / Wix / Zapier / Airtable
        - Wireframing => maquette du projet 
        - Planning prévisionnel du projet => Diagramme de GANTT
        - Liste des fonctionnalités du site (Users Cases)
        - Sécurité / RGPD 
        - Organisé votre base de données (MCD : Modèle Conceptuel des Données)
        - SEO (Search Engine Optimization) le fait d'être visible dans les pages de résultat de Google

Annexes => le devis => le prix de la prestation 

# Cas pratique

- en utilisant l'extension DRAWIO sur Visual Studio Code 
- créer une nouveau fichier `07-exo.dio`
- ce fichier va contenir un organigramme d'arborescence d'un site e commerce
- ce site e commerce contient les pages suivantes
    - front office 
        - page d'accueil
        - page de catégorie 
        - page produit
        - page du tunnel d'achat
            - page panier
            - page d'identification
            - page adresse de livraison / adresse de facturation
            - page de paiement
    - back office 
        - page de gestion du catalogue
        - page de gestion des commandes
        - page de gestion des réclamations
